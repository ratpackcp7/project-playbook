# Decision Records

Use ADRs to explain why important choices exist.

Create or update an ADR for changes to:

- ports, bind addresses, tunnels, proxies, UFW, Tailscale, Cloudflare
- Docker Compose resources, volumes, healthchecks, restart policies
- systemd units, timers, background workers
- persistent data paths, migrations, backups
- MCP tools, auth, allowlists, writable roots
- unusual configuration that future maintainers may question

Template: `ADR-template.md`
